1.  First, What's the Problem with a Normal JS Variable?

    1.  Let's say you tried doing this the "normal JavaScript way":

        const Header = () => {
        let btnName = "Login"; // normal JS variable

            return (
                <button onClick={() => { btnName = "Logout"; }}>
                {btnName}
                </button>
            );

        };

        What happens? : The variable does change internally — but React has no idea it changed. React won't re-render the component, so the button on screen stays as "Login" forever.

        The core problem: Normal JS variables don't trigger UI updates.

    2.  Enter useState() — The React Way

    const [btnName, setBtnName] = useState("Login");

    Think of this one line as doing **3 things at once:**

    | Part | What it is | Role |

        | `btnName` | State variable | Holds the current value |
        | `setBtnName` | Setter function | Updates the value AND re-renders UI |
        | `useState("Login")` | Hook call | Sets the **initial value** to `"Login"` |

    3.  Step 1 — Component loads for the first time:

            useState("Login")  →  btnName = "Login"

            result : The button renders and displays "Login".

        Step 2 - User clicks the button:

            onClick={() => {
            setBtnName("Logout");
            }}

            `setBtnName("Logout")` is called. Two things happen instantly:
            - `btnName` is updated to `"Logout"`
            - React **re-renders** the `Header` component

        Step 3 - Component re-renders

            useState("Login")  →  btnName = "Logout"  ← React remembers the new value

            The button now displays "Logout".

2.  Right now the button only goes Login → Logout, but never back.

         Here's the fix:
                onClick={() => {
                setBtnName(btnName === "Login" ? "Logout" : "Login");
                }}

    This checks the current value and flips it every time you click.

    Whenever you have data that changes over time and you want the UI to reflect that change, use useState.

4.  How React Changes a const Variable

    1.  The Confusion :

            1. Normally in JavaScript, const means you can never reassign it:

                    const btnName = "Login";
                    btnName = "Logout"; // ❌ ERROR — Cannot reassign a const variable

            So how does React seem to "change" it?

    2.  The Secret — React is NOT changing the variable.

            This is the key insight:

            React doesn't change the const variable. It throws the old one away and creates a brand new const with the new value.

    3.  What Actually Happens?

        1.  Step 1 — First render:

                const [btnName, setBtnName] = useState("Login");
                // React creates → const btnName = "Login"
                // This const lives for THIS render only

        2.  Step 2 — You click the button, setBtnName("Logout") is called:

                // React does NOT do this ❌
                btnName = "Logout"; // this would break const rules

                // React instead says:
                // "Okay, I need to re-render this component with new data"

        3.  Step 3 — React re-renders the ENTIRE component function from scratch:

                // The old render is GONE
                // A completely fresh new render starts
                const [btnName, setBtnName] = useState("Logout");

                // React creates a BRAND NEW const btnName = "Logout"


                The old `const btnName = "Login"` is **destroyed**, and a new `const btnName = "Logout"` is **born**.

                ###  Visual of What's Happening

                ────────────── RENDER 1 ──────────────
                const btnName = "Login"   ← created
                button clicked!
                const btnName = "Login"   ← DESTROYED
                ──────────────────────────────────────

                ────────────── RENDER 2 ──────────────
                const btnName = "Logout"  ← NEW const created fresh
                button clicked!
                const btnName = "Logout"  ← DESTROYED
                ──────────────────────────────────────

                ────────────── RENDER 3 ──────────────
                const btnName = "Login"   ← NEW const created fresh
                ...and so on
                ──────────────────────────────────────
                Each render is a completely fresh function call with its own brand new const.
